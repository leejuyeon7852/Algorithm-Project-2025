## 서울 지하철 경로 탐색 프로젝트

### 1. 프로젝트 소개
이 프로젝트는 서울교통공사의 실제 데이터를 활용해서 최적경로를 구현하는 알고리즘 시스템이다. 
단순 최단거리가 아니라 환승시간, 혼잡도를 함께 고려한 탐색을 수행한다.

알고리즘 수업의 기말과제이며, Dijkstra와 A* 알고리즘을 실제 데이터에 적용하였다.

### 2. 문제 정의
**문제**: 서울 지하철에서 출발역과 도착역이 주어졌을 때, 환승 시간과 혼잡도를 고려하여 사용자의 목적에 맞는 경로를 탐색하는 문제
1. 최단 시간 경로 탐색: 순수 이동시간과 환승 시간을 고려하여 가장 빠른 경로 계산
2. 혼잡도 고려 경로 탐색: 혼잡도를 고려하여 시간을 최소화 하는 경로 계산
3. 알고리즘 성능 비교: Dijkstra 알고리즘과 A* 알고리즘의 탐색 효율성 비교 분석

### 3. 사용 알고리즘
+ **Dijkstra알고리즘**

    출발역에서 모든 역까지의 최단거리를 계산하는 알고리즘으로 이동시간과 환승 페널티를 가중치로 사용하여 최단시간 경로를 탐색한다.

+ **A*알고리즘**

  목적지가 정해진 최단 경로 탐색 알고리즘으로 지하철역 좌표를 기반으로 직선 거리를 계산한 휴리스틱을 사용하여 탐색 효율을 향상 시킨다.

### 4. 실행 방법
1. main.py

   실행 방법
   
   ```python main.py```

   + 실행 시 최단경로를 기준으로 경로 추천하는 모드가 있고, 혼잡도를 기반으로 경로를 추천하는 모드가 있다. 
   + 모드 입력 후 출발역과 도착역 입력 후 경로가 출력되고, 그래프가 나타난다.
3. main2.py

   Dijkstra 알고리즘과 A* 알고리즘의 성능을 비교하는 프로그램이다.

   실행 방법

   ```python main2.py```
   + 실행 시 출발역과 도착역을 입력하면 두 알고리즘의 실행시간을 비교한 그래프가 나타난다.
